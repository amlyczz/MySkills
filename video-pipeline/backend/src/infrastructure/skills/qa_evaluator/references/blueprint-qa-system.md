### 评分方法：从 100 分起扣
起始 100 分。发现每项缺陷即扣分。

### 扣分规则：
1. **Element Completeness（元素完整性）**（扣 0-20 分）：
   - -0：每个场景都有丰富、类型正确的 ElementConfig 树
   - -5：一个场景元素稀疏或缺失
   - -10：多个场景的元素树类似占位符
   - -20：所有场景元素缺失或过于简单

2. **Animation Quality（动画质量）**（扣 0-20 分）：
   - -0：所有动画元素有正确的 inFrame/outFrame，列表使用了 stagger，有合适的 easing
   - -5：一两个元素缺少 outFrame
   - -10：列表元素没有 stagger，只有通用动画
   - -20：完全没有动画或明显错误

3. **Flex Layout Compliance（Flex 布局合规性）**（扣 0-15 分）：
   - -0：所有元素使用 position "flex-child"
   - -5：一个元素使用绝对定位
   - -10：多个元素使用绝对定位（有碰撞风险）
   - -15：布局系统根本性损坏

4. **Safe Exit OutFrame（安全退出帧）**（扣 0-15 分）：
   - -0：每个动画元素：outFrame = durationInFrames - 15
   - -5：一两个元素缺少 outFrame
   - -10：大多数元素缺少 outFrame（元素无法干净退出）
   - -15：完全没有设置 outFrame（场景过渡时视觉混乱）

5. **Subtitle Quality（字幕质量）**（扣 0-15 分）：
   - -0：按标点符号（句子）拆分 token，帧范围合理
   - -5：部分 token 过短（句子中间拆分短句）
   - -10：检测到逐词拆分（快速闪烁风险）
   - -15：没有字幕或帧范围完全错误

6. **Visual Cohesion（视觉一致性）**（扣 0-15 分）：
   - -0：主题一致，过渡得当，配色方案统一
   - -5：一处突兀的过渡或配色不匹配
   - -10：场景间视觉语言不一致
   - -15：没有统一的视觉风格

### 锚定案例：
- **90 分**：丰富的元素树、带 stagger 的 spring 动画、全部 flex-child、outFrame 正确、句子级字幕、统一的暗色霓虹主题。可投入生产。
- **70 分**：元素尚可但 30% 缺少 outFrame、一个场景使用绝对定位、字幕为短句级别。需要改进。
- **50 分**：稀疏的元素树、没有动画、到处是绝对定位、没有字幕。需要大幅重新生成。

### 关键规则：
- 默认状态是"找问题"。关注哪里出了问题，而不是哪里没问题。
- 一个"乍一看还行"的 Blueprint 是 60-70 分，不是 80 分。
- 只有当每个维度都确实合格时才给 85+ 分。
- 如果无法验证某个字段（如未设置 outFrame），按该维度最高扣分处理。
