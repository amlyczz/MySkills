### 事实核查协议：
你已获得来自实际仓库的源材料（SOURCE MATERIALS）。请使用它们来验证脚本中的技术声明。

对于脚本中的每个技术声明：
1. **识别声明** — 例如 "使用 Transformer 架构"、"基于 React 构建"、"比 X 快 3 倍"
2. **交叉比对** — 检查该声明是否得到以下材料的支持：
   - README 内容（明确提及）
   - 目录结构（例如，声明说 "微服务" 但只有一个 `src/` 文件夹，那就值得怀疑）
   - 依赖栈（例如，声明说 "基于 FastAPI 构建" 但 pyproject.toml 中列出的是 Flask）
   - 源代码片段（例如，声明说 "核心算法使用 A*" 但代码中实际是 Dijkstra 算法）
3. **对声明进行分类**：
   - **VERIFIED（已验证）**：源材料支持该声明
   - **UNVERIFIABLE（无法验证）**：源材料既不能确认也不能否认（中性——不扣分）
   - **CONTRADICTED（矛盾）**：源材料明确与该声明相矛盾 → 扣 15-25 分
   - **HALLUCINATED（幻觉）**：声明很具体但完全找不到证据 → 扣 10-20 分

重要细节：
- 像 "使用现代架构" 这样的声明过于模糊，无法验证——跳过
- 像 "使用 4 层 Clean Architecture" 这样的声明可以通过目录结构进行验证
- 没有证据不代表证据不存在——只标记 CONTRADICTED 或明显 HALLUCINATED 的声明
- 有疑问时，标记为 UNVERIFIABLE（不扣分）
