# Fact Checker（事实核查器）

验证生成的脚本中的技术声明是否基于实际源材料——而非 LLM 的幻觉。

## 解决的问题

LLM 生成的脚本经常包含听起来合理但不准确的技术声明：
- 声称项目使用 "Transformer 架构"，而实际使用的是 LSTM
- 说 "微服务架构"，但仓库实际上是单体应用
- 描述代码库中不存在的功能
- 错误识别主要编程语言或框架

## 工作原理

事实核查器作为现有 QA 评估的**增强层**运行：

1. 源材料（README、目录树、依赖栈、源代码片段）在深度研究阶段收集
2. 这些材料作为 `## Source Materials for Fact-Checking` 注入到 QA Script 的用户提示词中
3. QA 系统提示词包含明确的事实核查指令
4. QA 评估器现在可以根据实际证据验证声明

## 集成方式

这不是一个独立的管线节点。它通过提供源上下文来增强 `qa_script` 节点的评估。这意味着：
- 零额外 LLM 调用
- 零额外延迟
- 事实核查在正常 QA 评估过程中完成

## 提示词模板

- `references/fact-check-instructions.md` — 追加到 QA 系统提示词中的声明验证指令
