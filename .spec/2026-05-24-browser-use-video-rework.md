# 2026-05-24 Browser Use 视频重构与视觉优化 Spec

## 问题陈述
现有的 `browser-use/browser-use` 视频生成结果（位于 `output/github/.../browser-use-v2`）存在严重缺陷：
1. **内容逻辑缺失**：未遵循 `repo-analyzer` 的 4-Phase 工作流。缺乏真实素材下载（如截图、视频），缺乏深度源码分析。视频总时长不足，未达到 3+ 分钟。内容结构未按照“介绍用处 -> 适用场景 -> 安装启动 -> 源码深度解析 -> 总结与结构拆解”的科普教学逻辑展开。
2. **视觉 UI 崩坏**：
   - 首尾场景和中间存在莫名的光条（可能是背景或渐变组件未正确渲染）。
   - 标题组件展示为竖条排版，且与其他元素严重重叠。
   - 下方的卡片/悬浮效果与底部字幕相互遮挡。
   - 字幕超出屏幕边界被截断。
   - 章节 2 代码块样式丑陋且偏左，需替换为 Mac 原生代码块样式（参考 remotion-best-practices）。
   - 章节 2 右侧的“设计精要” callout 组件过于丑陋，需删除。
   - 场景过渡（Crossfade）导致新旧场景内容重叠混乱。
   - 章节 3 卡片图标丑陋。
   - 表格组件样式丑陋。
   - 最后场景也存在莫名光条。

## 设计方案

### 1. 内容脚本重构 (严格遵循 repo-analyzer 规范)
重新规划 3+ 分钟的视频结构：
- **第一部分：引言与核心用处**：介绍 Browser Use 是什么（让 AI 像人一样浏览网页的开源框架），适用场景（数据采集、自动化测试、AI 助手）。配合真实的网页自动化执行录屏。
- **第二部分：快速起步与安装**：如何通过 pip 安装，只需 6 行代码的快速启动演示。
- **第三部分：源码深度解析（核心）**：
  - 核心四层架构分析（Agent -> Browser -> Actor -> Watchdog）。
  - 重点解读 `Agent.step()` 核心循环（看 _prepare_context -> 想 _get_next_action -> 做 _execute_actions -> 记 _post_process）。
  - 需要配合架构图展示。
- **第四部分：生态与总结结构拆解**：20+ LLM 后端支持，MCP 协议，结构拆解总结。呼吁 Star。

### 2. UI/UX 视觉重构方案
- **清理不明光条**：排查并移除导致全屏或局部莫名光条的容器（如 `gradient-text`, `badge`, `luxury-card` 背景错乱）。
- **排版与遮挡修复**：
  - 将所有全角/长标题改为横向展示（设置最小宽度，确保文字不换行成竖条）。
  - 重新分配布局区域。字幕严格限制在底部 `y: 85%`，主内容区域限制在上方，确保互不遮挡。
- **字幕截断修复**：为字幕组件增加最大宽度限制，开启文字换行。
- **Mac 原生代码块**：实现 Mac 原生风格（左上角红黄绿三个小圆点）的代码块 UI。代码块居中展示。删除原有的“设计精要”。
- **卡片与表格重绘**：废弃原有的 `icon-grid` 和 `comparison-table`，替换为更现代化、扁平化的卡片设计或直观对比。
- **转场优化**：废弃简单的全屏 Crossfade，通过调整各元素动画出入场时间点，保证上个场景元素退出后再展示新场景，避免图层重叠。

## 验收标准
1. **内容**：包含真实的截图/录屏素材。内容逻辑清晰，包含深度源码讲解。
2. **视觉**：无重叠、无越界、无竖向错乱标题、无莫名光条。代码块呈现 Mac 原生风格且居中。卡片与对比展示美观。转场平滑。
3. **流程**：获得用户 Review 批准后，执行修改。
