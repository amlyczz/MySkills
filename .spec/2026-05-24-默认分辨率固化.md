# 默认分辨率固化 — 16:9 1080p 30fps

> 视频默认参数写死，不可由 LLM 配置。

---

## 现状

`globalSettings` 中 `resolution` 和 `frameRate` 由 Blueprint JSON 决定：

```ts
globalSettings: {
  frameRate: 30,
  resolution: { width: 1920, height: 1080 },
}
```

LLM 可以输出任意值（如 720p、竖屏、60fps），导致渲染异常。

## 方案

1. `Blueprint` 类型中移除 `frameRate` 和 `resolution` 字段
2. 这两个值由引擎常量锁定：

```ts
// engine/constants.ts
export const DEFAULT_FPS = 30;
export const DEFAULT_WIDTH = 1920;
export const DEFAULT_HEIGHT = 1080;
```

3. `Root.tsx` 中所有 Composition 统一使用常量
4. `TemplateRenderer` 不再从 blueprint 读取这两个值
5. Blueprint JSON Schema 文档中删除这两个字段

## 影响范围

| 文件 | 变更 |
|------|------|
| `engine/types.ts` | 移除 `frameRate` / `resolution` |
| `engine/constants.ts` | 🆕 常量定义 |
| `Root.tsx` | 全局用常量替换硬编码的 FPS=24/WIDTH/HEIGHT |
| `engine/TemplateRenderer.tsx` | 移除对 blueprint 中这两个字段的读取 |
| 所有 blueprint 文件 | 移除 `frameRate` / `resolution` 字段 |
| `contracts/component-props-schema.json` | 移除对应文档 |

## 验收标准
- [x] 所有 Composition 统一 1920x1080 @ 30fps
- [x] Blueprint JSON 中不再出现 `frameRate` / `resolution`
- [x] LLM 无法通过 JSON 改变分辨率或帧率
