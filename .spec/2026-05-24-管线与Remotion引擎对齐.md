# 管线与 Remotion 引擎对齐

> 修复 3 个断裂点，让 pipeline-orchestrator 能正常驱动当前 Remotion 引擎。

---

## 背景

当前 Remotion 引擎已全面升级为 Blueprint JSON 驱动（10 维度、59 个注册组件、12 项工业化补强）。但 pipeline-orchestrator 仍在使用旧版 catalog 和旧的 `video_config.json` 格式，二者完全不兼容。

---

## 断裂点 1：缺少 `VideoComposer` 通用入口

**现状**：`Root.tsx` 只有 15 个绑死 blueprint 的 Composition（如 `DarkNeon-Blueprint`、`Memphis-Blueprint`）。管道需要调用 `npx remotion render VideoComposer`，但没有这个 Composition。

**修复**：新增 `VideoComposer`，从 props 接收 Blueprint JSON：

```tsx
// Root.tsx
<Composition
  id="VideoComposer"
  component={VideoComposer}
  defaultProps={{ blueprintJson: "{}" }}
  durationInFrames={1}
  fps={DEFAULT_FPS}
  width={DEFAULT_WIDTH}
  height={DEFAULT_HEIGHT}
  calculateMetadata={calculateVideoComposerMetadata}
/>
```

`VideoComposer` 组件：
1. 接收 `blueprintJson` prop（JSON 字符串）
2. `JSON.parse` 得到 `Blueprint` 对象
3. 传递给 `TemplateRenderer`
4. `calculateMetadata` 用 `calculateTotalFrames` 动态算时长

管道调用：
```bash
npx remotion render VideoComposer --props='{"blueprintJson":"$(cat output/.../blueprint.json | jq -c .)"}'
```

## 断裂点 2：`ScriptTimelineComposer` 空目录

**现状**：`script-timeline-composer/` 没有文件。管道期望它输出 `video_config.json` + `timeline.json`。

**修复**：创建 `script-timeline-composer/skill.md`，让 AI Agent 输出 Blueprint JSON 而非旧的 video_config：

```
输出：
  output/{date}/{repo}/
    ├── blueprint.json     ← 🆕 直接输出 Blueprint（替代 video_config.json + timeline.json）
    ├── voiceover.mp3      ← 由 MediaGenerator 生成
    └── bgm.mp3            ← 由 MediaGenerator 生成
```

Skill 核心指令：
- 读取 RepoAnalyzer 的 content.json
- 按 Blueprint Schema 输出完整 JSON
- 场景结构用 pipeline 定义的结构模板（hook→problem→solution→showcase→features→cta）
- 组件类型只能选用 `contracts/component-props-schema.json` 中注册的
- 动效类型只能选用 `AnimationType` 枚举值
- 背景类型只能选用 `BackgroundType` 枚举值
- 分辨率/帧率已写死，不要输出
