# 组件复用体系补全 — 基于 Remotion Reusability 模式

> 分析 Remotion 官方 reusability 文档，对照现有引擎，识别缺失的通用组件。

---

## 1. Remotion Reusability 核心模式

来源：https://www.remotion.dev/docs/reusability

| 模式 | 含义 | 当前引擎状态 |
|------|------|------------|
| **Encapsulation** | 视频逻辑封装为独立组件 | ✅ componentRegistry + Dumb Component |
| **Props-based reuse** | 通过 props 定制每个实例 | ✅ ElementConfig.props |
| **Sequence wrapping** | 用 `<Sequence>` 限制时长和偏移 | ✅ TransitionSeries |
| **layout="none"** | Sequence 内联而非绝对定位 | ⚠️ 未在引擎中使用 |

## 3. 与 10 维度的对应

```
D1 色彩      → Title, LowerThird, CtaButton, StatCard, QuoteCard, Badge
                所有颜色通过 var(--color-*) 变量
D2 排版      → Title (text-h1/h2/h3 工具类), NumberCounter
D3 布局      → IconGrid, SplitMedia (Flex/Grid 纯布局)
D4 形态      → CtaButton (radius/shadow 变量), LowerThird
D5 动效      → NumberCounter (数字滚动微动效), KenBurns (缩放微动效)
D7 媒体      → SplitMedia (Img/Video 集成), KenBurns (图片驱动)
D9 容错      → Title (fitText), LowerThird (文本溢出防护)
D10 注册     → 全部注册进 componentRegistry
```

### Title 组件详细设计 （Remotion 文档直接提及的复用模式）

```
props:
  level    → "h1" | "h2" | "h3"
  text     → 标题文本
  subtitle → 可选副标题（D9: fitText 防溢出）
  align    → "left" | "center" | "right"
  maxWidth → 最大宽度（触发 fitText）

D9: 内置 fitText() 计算最佳字号
D1: 颜色通过 var(--color-primary) / var(--color-foreground)
D2: 字号通过 globalSettings.typography.scales 变量
```

### LowerThird 组件详细设计

```
props:
  name     → 人名/标题
  subtitle → 职位/补充信息
  position → "bottom-left" | "bottom-right"
  showBar  → 是否显示左侧装饰条

D4: 圆角/投影通过 var(--radius-*) / var(--shadow-*)
D1: 背景/文字色通过 var(--color-surface) / var(--color-foreground)
```

## 4. 实施计划
