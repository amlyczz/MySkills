## 1. 静态基础变换 (Base Transform & Style)

**痛点**：当前 `layout` 只有 x/y/width/height。想放一个"天生旋转 45° 且半透明"的元素，只能把旋转塞进 props，不干净。

**方案**：

```ts
export interface ElementConfig {
  layout?: {
    position?: "absolute" | "relative" | "flex-child";
    x?: number | string;
    y?: number | string;
    width?: number | string;
    height?: number | string;
    zIndex?: number;
    // 🆕 静态变换（不依赖帧动画）
    scale?: number;           // 默认 1
    rotation?: number;        // 度数，如 45
    opacity?: number;         // 0-1，默认 1
  };
  // 🆕 裸 CSS 覆盖（Tailwind 无法表达的动态值）
  style?: React.CSSProperties;
}
```

**与 animation 的区别**：
- `layout.rotation: 45` → 这个元素**始终**是 45°，不进动画轨道
- `animation: { type: "fade-up", ... }` → 在指定的 inFrame~outFrame 期间从无到有变化

**引擎行为**：`ElementRenderer` 合并动画输出到最终 style 时，静态 layout 值作为基线：
```
最终 transform = layout.rotation(静态) + animation.transform(动态叠加)
最终 opacity    = layout.opacity(静态) * animation.opacity(动态叠加)
```

**LLM 示例**：
```json
{
  "type": "badge",
  "props": { "text": "NEW" },
  "layout": { "position": "absolute", "x": 20, "y": 20, "rotation": -15, "opacity": 0.85 }
}
// → Badge 始终在 (20,20)，始终旋转 -15°，始终 0.85 透明度，无需动画
```

## 2. 循环动效 (Looping Animation)

**痛点**：`AnimationConfig` 是完美的 A→B 单向动效。但视频里常见呼吸灯、持续浮动——这些是 A→B→A→B 的循环。

**方案**：

```ts
export interface AnimationConfig {
  // ... 现有字段不变
  loop?: {
    type: "pulse" | "float" | "spin" | "wiggle";
    durationInFrames: number;  // 一个循环周期
    amplitude?: number;        // pulse/float 的幅度 (0-1)
  };
}
```

**各类型效果**：
| loop.type | 效果 | 实现 |
|-----------|------|------|
| `pulse` | 缩放到 1+amplitude 再缩回来 | `scale = 1 + sin(frame) * amplitude` |
| `float` | 上下浮动 | `translateY = sin(frame) * amplitude * 20px` |
| `spin` | 持续旋转 | `rotateZ = (frame % durationInFrames) / durationInFrames * 360` |
| `wiggle` | 左右摇摆 | `rotateZ = sin(frame) * amplitude * 10°` |

**引擎行为**：`applyAnimation()` 中，`loop` 和入场 `animation.type` 并行不悖：
- 入场动效结束后，loop 持续运行
- `outFrame` 触发后，loop 随元素一同消失

**LLM 示例**：
```json
{
  "type": "organic-blob",
  "animation": {
    "type": "scale-bounce",
    "timeline": { "inFrame": 0 },
    "loop": { "type": "pulse", "durationInFrames": 120, "amplitude": 0.08 }
  }
}
// → 入场弹跳到位后，持续以 4 秒周期呼吸缩放
```

## 3. 模板变量 Schema (Variables for UI Forms)

**痛点**：Blueprint 是确定的最终渲染状态。但"模板"对外暴露哪些可填参数？前端需要这个元信息来生成表单。

**方案**：

```ts
export interface Blueprint {
  meta: { id, name, description? };
  // 🆕 模板暴露的动态参数（前端据此生成表单面板）
  variables?: {
    content: Array<{
      key: string;          // 变量名，如 "productName"
      label: string;        // 表单标签，如 "Product Name"
      type: "string" | "number" | "image" | "textarea";
      default: unknown;
    }>;
    theme: Array<{
      key: string;          // 如 "primaryColor"
      label: string;        // 如 "Primary Color"
      type: "color" | "font";
      default: unknown;
    }>;
  };
  globalSettings: { ... };
  // ...
}
```

**使用场景**：
- AI 生成模板时，同时声明哪些字段是可配置的
- 前端按 `variables` 自动渲染输入表单（文本框/颜色选择器/图片上传）
- 用户填写后，将值注入 `globalSettings` 或 `data` → 渲染

**LLM 示例**：
```json
{
  "meta": { "id": "product-hunt-launch", "name": "Product Launch Video" },
  "variables": {
    "content": [
      { "key": "productName", "label": "Product Name", "type": "string", "default": "MyProduct" },
      { "key": "heroImage",   "label": "Hero Image",    "type": "image",  "default": "" },
      { "key": "price",       "label": "Price",          "type": "string", "default": "$29/mo" }
    ],
    "theme": [
      { "key": "primaryColor", "label": "Primary Color", "type": "color", "default": "#4285F4" },
      { "key": "fontFamily",   "label": "Font",           "type": "font",  "default": "Inter" }
    ]
  }
}
// → 前端渲染出 3 个输入框 + 2 个颜色/字体选择器
```

## 4. 音频降噪 / Ducking

**痛点**：旁白响起时 BGM 应该自动降低音量，否则人声被音乐盖住。目前没有这个机制。

**方案**：

```ts
globalSettings: {
  audio?: {
    bgmUrl?: string;
    bgmVolume?: number;
    sfx?: Record<string, string>;
    // 🆕 旁白响起时 BGM 自动降量
    ducking?: {
      enabled: boolean;
      duckToVolume?: number;    // BGM 降到多少（默认 0.1）
      fadeDurationFrames?: number;  // 淡入淡出帧数（默认 10）
    };
  };
}
```

**引擎行为**：`TemplateRenderer` 检测 `ducking.enabled === true` → 扫描所有 `scenes[].voiceover` → 生成 BGM volume 插值曲线：

```
BGM volume:
  voiceover.startFrame - fadeDuration → 开始从 bgmVolume 降到 duckToVolume
  voiceover.startFrame → 降到 duckToVolume
  voiceover.endFrame → 开始从 duckToVolume 升回 bgmVolume
  voiceover.endFrame + fadeDuration → 恢复到 bgmVolume
```

Remotion 的 `<Audio>` 支持 `volume` 回调函数 `(frame) => number`，引擎根据所有 voiceover 的时间线计算出逐帧的 BGM volume 值。

**LLM 示例**：
```json
{
  "globalSettings": {
    "audio": {
      "bgmUrl": "https://example.com/bgm.mp3",
      "bgmVolume": 0.6,
      "ducking": { "enabled": true, "duckToVolume": 0.1, "fadeDurationFrames": 10 }
    }
  }
}
// → 每当任何 scene 的 voiceover 播放时，BGM 自动在 10 帧内降到 0.1
// → 旁白结束后 10 帧内恢复到 0.6
```

## 总结：4 项与前面 8 项的去重

这 4 项全部是**新角度**，与前面 8 项无重叠：

| # | 增强项 | 与前面 8 项的关系 |
