# Blueprint 工业化补强 — 8 项增强方案

> 目标：从「人工辅助渲染」升级到「LLM 吐出即渲染，无需人工修图」。

---

## 1. 动画编排：子元素交错入场 (Stagger)

**问题**：当前 `AnimationConfig` 只作用于单个元素。LLM 需要为 10 个列表项手算 10 个不同 `inFrame`，极易出错。

**方案**：`AnimationConfig` 新增 `stagger` 字段：

```ts
export interface AnimationConfig {
  // ... 现有字段
  stagger?: {
    delayPerChild: number;          // 子元素间延迟帧数
    direction?: "forward" | "reverse";  // 正向/反向
  };
}
```

**引擎行为**：`ElementRenderer` 递归渲染 children 时，检测父元素的 `stagger` 配置，自动为第 i 个子元素计算 `inFrame = parent.inFrame + i * stagger.delayPerChild`（reverse 时反向）。

**LLM 示例**：
```json
{
  "type": "split-layout",
  "animation": {
    "type": "fade-up",
    "timeline": { "inFrame": 10 },
    "stagger": { "delayPerChild": 8, "direction": "forward" }
  },
  "children": [
    { "type": "stat-card", "props": { "value": "12K", "label": "Users" } },
    { "type": "stat-card", "props": { "value": "98%", "label": "Uptime" } },
    { "type": "stat-card", "props": { "value": "4.9", "label": "Rating" } }
  ]
}
// → 三张卡片依次在 frame 10, 18, 26 入场
```

## 2. 全局动效 Token (Motion Design Tokens)

**问题**：LLM 重复输出相同的 easing 参数（如 `spring(1,14,100)`），不一致时破坏视频整体感。

**方案**：`globalSettings` 新增 `motionTokens`：

```ts
globalSettings: {
  motionTokens?: Record<string, MotionToken>;
}

type MotionToken = {
  easing: { type: "spring"; params: { mass: number; damping: number; stiffness: number } }
       | { type: "bezier"; bezier: [number, number, number, number] };
  duration?: number;
};
```

**引擎行为**：`AnimationConfig.easing` 可以是 Token 名字符串 → 引擎查表替换为完整参数。

**LLM 示例**：
```json
{
  "globalSettings": {
    "motionTokens": {
      "gentle": { "easing": { "type": "spring", "params": { "mass": 1, "damping": 14, "stiffness": 100 } }, "duration": 30 },
      "snappy": { "easing": { "type": "spring", "params": { "mass": 0.8, "damping": 10, "stiffness": 200 } }, "duration": 20 },
      "smooth":   { "easing": { "type": "bezier", "bezier": [0.16, 1, 0.3, 1] }, "duration": 45 }
    }
  },
  "scenes": [{
    "elements": [{
      "animation": { "type": "fade-up", "easing": "gentle" }
    }]
  }]
}
// "gentle" → 引擎查表 → spring({ mass:1, damping:14, stiffness:100 })
```

## 3. 数据驱动组件：Props Schema 文档化

**问题**：`ElementConfig.props: Record<string, unknown>` 对 LLM 太宽松，容易输出无效 props。

**方案**：不改变 TS 类型（保持灵活性），但在 System Prompt 中提供核心组件的 props 契约说明。同时在 `contracts/` 中新增 `component-props-schema.json`：

```json
{
  "data-bar-chart": {
    "props": {
      "data": { "type": "array", "items": { "label": "string", "value": "number", "compareValue?": "number", "color?": "string" } },
      "maxValue": { "type": "number | 'auto'" },
      "showComparison?": "boolean",
      "isHighlight?": "boolean"
    }
  },
  "pricing-card": {
    "props": {
      "data": {
        "title": "string",
        "slogan": "string",
        "features": "string[]",
        "price": "string",
        "accentColor": "string"
      }
    }
  }
}
```

**引擎行为**：渲染前用 Zod 校验 `props`，不合规时返回清晰错误 → LLM 可自我修正。

## 4. 条件渲染 (Conditional Elements)

**问题**：无法根据变量切换场景/元素。如 `showComparison=true` 时显示对比图表。

**方案**：`ElementConfig` 新增 `condition` 字段：

```ts
export interface ElementConfig {
  condition?: string;  // 如 "showComparison == true"
}
```

**引擎行为**：`ElementRenderer` 检测 `condition` → 查找 `globalSettings.data` 或 `scene.props` 中的变量 → 安全求值 → false 时跳过渲染。

**LLM 示例**：
```json
{
  "globalSettings": {
    "data": { "showComparison": true, "planTier": "pro" }
  },
  "scenes": [{
    "elements": [
      { "type": "data-bar-chart", "props": { "showComparison": true } },
      { "condition": "planTier == 'pro'",
        "type": "quote-card", "props": { "quote": "Best investment ever.", "author": "CEO" } }
    ]
  }]
}
```

**安全约束**：只允许 `==`, `!=`, `>`, `<`, `>=`, `<=`, `&&`, `||` 操作符，用 `new Function()` 沙箱求值。

## 5. 响应式/安全区域

**问题**：只有一个分辨率，不支持 9:16 竖屏或安全边距。

**方案**：`globalSettings` 新增：

```ts
globalSettings: {
  safeArea?: {
    top: number; right: number; bottom: number; left: number;
    unit: "px" | "%";
  };
}
```

**引擎行为**：`TemplateRenderer` 将 `safeArea` 注入为 CSS 变量 `--safe-top` / `--safe-right` / `--safe-bottom` / `--safe-left`，组件通过 Tailwind `pt-[var(--safe-top)]` 使用。

## 6. 音频高级控制

**问题**：`VoiceoverConfig` 缺 `endFrame`/`loop`；字幕缺 SRT URL 备选；SFX 缺全局帧模式。

**方案**：

```ts
export interface VoiceoverConfig {
  audioUrl: string;
  text: string;
  startFrame: number;
  endFrame?: number;      // 🆕 结束帧
  volume?: number;
  loop?: boolean;          // 🆕 循环播放
}

export interface SubtitleConfig {
  tokens?: SubtitleToken[];
  srtUrl?: string;        // 🆕 SRT 文件 URL，引擎自动解析
  captionsUrl?: string;    // 已有
  highlightColor?: string;
  fontSize?: number;
}

export interface SfxTrigger {
  sfx: string;
  atFrame: number;
  frameOf?: "scene" | "global";  // 🆕 默认为 scene
  volume?: number;
}
```

**引擎行为**：
- `srtUrl` → TemplateRenderer 用 `@remotion/captions` 的 `parseSrt()` 解析 → 生成 `SubtitleToken[]`
- `endFrame` → Sequence 提前截断 Audio
- `frameOf: "global"` → SFX 用全局帧偏移，不随场景重置

## 7. 全局数据绑定 (Template Variables)

**问题**：多个场景共享数据（产品名、价格），当前只能每个场景手写一遍。

**方案**：`Blueprint` 新增 `data` 字段 + props 模板语法：

```ts
export interface Blueprint {
  data?: Record<string, unknown>;  // 🆕 全局共享数据
  // ...
}
```

**LLM 示例**：
```json
{
  "data": { "productName": "Claude Code", "price": "$20/mo", "heroImage": "https://..." },
  "scenes": [
    { "elements": [
      { "type": "title", "props": { "text": "$data.productName" } }
    ]},
    { "elements": [
      { "type": "pricing-card", "props": { "data": { "price": "$data.price" } } }
    ]}
  ]
}
```

**引擎行为**：`TemplateRenderer` 在传递给 `ElementRenderer` 前，递归扫描所有 `props` 中 `"$data.xxx"` 模式，替换为 `blueprint.data.xxx` 的实际值。

## 8. Zod Schema 运行时校验

**问题**：LLM 输出的 JSON 可能含拼写错误、枚举值不对、字段缺失。

**方案**：基于当前 `types.ts` 生成 Zod Schema（放在 `contracts/blueprint-schema.ts`），`TemplateRenderer` 接收 props 时先 `blueprintSchema.parse(blueprint)`，校验失败时抛出结构化错误。

```ts
import { z } from "zod";

export const animationTypeEnum = z.enum([
  "none", "fade-in", "fade-out", "fade-up", "fade-down",
  "scale-in", "scale-bounce", "slide-left", "slide-right",
  "slide-up", "slide-down", "bar-grow", "typewriter",
]);
// ... 完整 schema
export const blueprintSchema = z.object({ ... });
```

**错误示例**：
```
Blueprint validation failed:
  scenes[0].elements[0].animation.type: "fadein" is not valid.
  Did you mean "fade-in"?
```

## 优先级排序

| # | 增强项 | 优先级 | 理由 |
|---|--------|--------|------|
| 1 | Stagger 编排 | **P0** | LLM 手算 inFrame 几乎不可能正确 |
| 2 | Motion Tokens | **P0** | 保证视频动效一致性，减少 LLM 重复输出 |
| 8 | Zod 校验 | **P0** | 最后防线，LLM 输出错误可自我修正 |
| 4 | 条件渲染 | **P1** | 解锁动态模板（plan tier / dark mode） |
| 7 | 数据绑定 | **P1** | 消除跨场景重复数据 |
| 3 | Props Schema 文档 | **P1** | 配合 Zod 校验提高 LLM 准确度 |
| 6 | 音频高级控制 | **P2** | 已基本可用，增强边界情况 |
| 5 | 响应式安全区 | **P2** | 单分辨率已覆盖当前场景 |

---

## 验收标准
- [x] Stagger: 父元素配置 `stagger` 后，子元素自动交错入场
- [x] Motion Tokens: `easing: "gentle"` 可替代完整 easing 配置
- [x] Zod 校验: 非法 Blueprint 返回可读错误而非崩溃
- [x] 条件渲染: `condition` 表达式的元素按预期显示/隐藏
- [x] 数据绑定: `$data.xxx` 被正确替换为全局值
