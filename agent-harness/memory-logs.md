# 持久记忆：AI 操作日志（.memory/）

主动维护 AI 操作日志，作为项目的"工程决策记忆"。

## 存储位置

- 所有日志文件放在 `.memory/` 目录下。
- 按日期划分文件：`.memory/YYYY-MM-DD.md`。
- 文件头部包含该日期的任务索引（可简要列出当天完成的条目标题）。

## 记录粒度

- 不能太短：一次文件格式化不用记录。
- 不能太长：一个大功能拆分为多个子任务时，每个子任务完成时追加一条记录。
- 必须记录的操作：任何符合"大功能"的任务、重要的技术决策、架构变更、测试策略调整、依赖变更、技术栈调整等。

## 日志条目模板（追加到当日文件末尾）

```markdown
## [15:30] 完成"用户登录"领域模型
- 类型：大功能-子任务
- 关联 Spec：.spec/2026-05-18-1530-用户登录.md
- 修改摘要：新增 User 实体，Password 值对象，UserRepository 接口。
- 决策：密码加密算法使用 bcrypt，在 infrastructure 中实现。
- 遗留问题：无
```

## 检索索引

- 每日文件内部条目按时间倒序，天然形成时间索引。
- 每月初 AI 可自动生成 `.memory/INDEX.md` 汇总各日期的关键操作，但非必须，用户主动要求时再生成。
- 搜索操作日志时，使用 `grep -r "关键词" .memory/`。
