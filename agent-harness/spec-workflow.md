# Spec 驱动工作流（Plan Mode）

当用户以"Plan Mode"、需求描述、重构或触发一个大功能等你认为需要写spec的情况，你必须按以下步骤执行，不得直接编码。

## 步骤

1. 阅读 `.spec/` 下的技术栈和最近的 spec 文件，理解上下文。
2. 生成 Spec 文件：
   - 路径：`.spec/YYYY-MM-DD-HHMM-<简短描述>.md`
3. 暂停，明确请求用户确认：
   "我已生成 Spec 草稿 `[文件名]`，请确认验收标准和技术方向，或提出修改意见。确认后我将进入 TDD 流程。"
4. 用户确认后，将状态改为 `confirmed`，并按照 TDD 流程执行。
5. **Spec 执行完毕后，按照 [`memory-logs.md`](memory-logs.md) 的规范，追加日志记录，包含：本次 Spec 实现的修改摘要、技术决策、遗留问题等信息。
