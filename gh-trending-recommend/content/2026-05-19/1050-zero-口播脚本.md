# 口播脚本 - 2026年05月19日 - zero

编程语言的语法设计，从来都是围绕人类的阅读习惯展开的——缩进要清晰、命名要有语义、括号要对称。但过去两年，真正高频写代码的"用户"正在悄然更替。当 Agent 已经成为代码的主要生产者，我们是不是应该重新想想：如果一门语言的读者不是人，而是 AI，它会是什么样？

Vercel Labs 四天前开源了一个叫 Zero 的项目，用一句话定义：这是第一个从 Day 1 就为 Agent 设计的编程语言。它不是在现有语言上套一层 AI 友好的壳，而是从编译器到标准库到工具链，全部围绕一个前提重建——代码的写作者和检查者是 Agent。

Zero 的设计有五个核心原则。第一，语言表面积极小且高度规整，Agent 靠几个示例和编译器的反馈就能学会，不需要理解人类的缩进美学。第二，标准库足够深，常见能力都内置于统一的 API 中，写程序的第一步不再是搜索依赖包。第三，工具链完全结构化——编译诊断、依赖图、体积分析、修复建议全部输出为 JSON，Agent 可以直接解析并执行下一步操作，不需要正则匹配错误信息。第四，支持直接编译为原生可执行文件，单文件部署，无运行时。第五，甚至内置了 Web 路由分析，`zero routes` 一条命令就能让 Agent 理解整个服务的 API 结构。

项目用 C 语言编写，Apache 2.0 开源，四天 2300 多 Star。目前还处于 pre-1 阶段，官方明确表示语法和 API 会持续变更，不建议用于生产环境。但它提出的问题非常值得思考：当 AI Agent 成为编程语言的第一公民，我们对"好语言"的定义会不会彻底改变？

如果你在构建 Agent 驱动的自动化流水线，Zero 的工具链设计思路——尤其是全结构化输出这个理念——值得借鉴。
