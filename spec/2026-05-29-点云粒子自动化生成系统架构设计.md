# 2026-05-29-实验性点云粒子系统Demo

## 问题陈述
为实现“点云粒子自动化生成系统”的渲染能力，需要探索验证 Remotion + R3F + Three.js 方案。为了既不增加孤立的外置工程依赖成本，又能避免影响现有系统逻辑，决定将此实验性 Demo 内嵌到现有的 `video-pipeline/frontend/remotion` 工程内。通过将其实现为一个完全独立的子组合（Composition），确保其开发和运行在物理上共用依赖，在逻辑上完美隔离。

## 设计方案

### 1. 依赖追加（共用宿主工程）
在 `video-pipeline/frontend/remotion/package.json` 内安装所需的实验性渲染库：`three`, `@remotion/three`, `@react-three/fiber`, `@react-three/drei`, `@react-three/postprocessing`。保证依赖由 `npm install` 统一管理。

### 2. 独立功能模块（非侵入）
在现有的 `src/` 下新建 `ParticleDemo/` 目录，封装全部实验逻辑：
- `config.ts`: 纯硬编码配置（包含至少 2 种特效预设和 2 种相机运镜参数），代替外部系统传入的契约数据。
- `CameraRig.tsx`: 将时间 `t = frame / fps` 转化为独立的镜头轨迹方程。
- `ParticleSimulation.tsx`: 以无状态方程 (Stateless GLSL Formula) 取代常规的有状态 Compute Shader，从根源解决 Remotion 时间轴多点乱序跳跃带来的坐标错位问题。配合 `<points>` 和 Bloom 特效实现仙气质感渲染。

### 3. 入口与自动化（隔离运行）
- 在现有的 `Root.tsx`（或对应入口）中追加注册一个独立的 `<Composition id="ParticleDemo" ... />`，原有的任何 Composition 都不作修改。
- 增加一个专属的 `scripts/render-particle-demo.mjs`。独立执行 `@remotion/bundler` 批量调用渲染流程，仅输出实验性质的粒子视频，彻底与主业务线解耦。

## 验收标准
1. **工程整洁度**：除了在 package.json 增加依赖和 Root 注入新的 Composition 之外，没有修改现存的任何组件与逻辑文件。
2. **确定性渲染**：针对 `ParticleDemo`，拖动时间轴进度条时粒子没有随机抖动或错乱闪烁，无状态粒子计算方式被证明可行。
3. **视觉效果**：泛光 (Bloom) 和混合 (AdditiveBlending) 正确起效。
4. **一键渲染验证**：能够单独运行该 Demo 的专属 Node 脚本并正确出片。
