# 口播脚本 - 2026年05月20日 - agents-best-practices

5 天时间，854 个 Star，一个没有一行可执行代码的仓库在 GitHub 上跑出了这个增速。它叫 agents-best-practices，不提供任何程序或框架，却覆盖了构建 AI Agent 所需的完整运行时纪律。

这个项目的核心命题只有一句话：模型提出动作，Harness 负责验证、授权、执行和记录。也就是说，当你让一个 Agent 去读写数据库、调用 API 或者发送消息时，真正执行操作的不应该是模型本身，而是你构建的外围控制层。这本质上是一个从"信任模型输出"到"验证模型输出"的架构范式切换。

项目以 Agent Skill 的形式发布，兼容 Codex、Claude Code 和任何支持 SKILL.md 规范的运行时。它的内容结构非常清晰——15 份参考文档，从最简 MVP 的蓝图设计，到循环预算和终止机制，再到工具权限分级、上下文压缩、提示词缓存策略、安全评估和上线 Checklist，覆盖了一个生产级 Agent 从原型到上线的全生命周期。

举几个实际问题来说明它的价值。如果你的 Agent 会陷入无限循环，它建议加 step、time、cost 三重预算；如果上下文压缩后丢失了审批状态，它建议把活跃状态持久化到 prompt 之外；如果你打算暴露一个通用的 read_write 工具给模型，它会直接告诉你不要这么做——换成窄类型的 typed tool，每次调用带上结构化结果和确定性权限检查。

值得注意的是，这个项目的适用范围并不局限于编程 Agent。文档明确指出，研究、客服、运维、销售、法务甚至医疗流程中的 Agent，都需要同一套核心运行时纪律。而它推荐的演进路径是：先用单 Agent MVP 验证核心循环，只在有数据支撑的失败出现后，才逐步引入子 Agent、目标循环和更广泛的自主性。
